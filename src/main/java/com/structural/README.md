# 结构型模式 
结构型模式可以描述两种东西：类结构 对象结构；
类结构：关心类的组合。 继承，实现；
对象结构：关心类与对象的组合。关联关系；

## 适配器模式
让那些不兼容的类一起工作；

## 桥接模式
将抽象部分与实现部分解耦，使两者可以独立变化；

## 组合模式
组合多个对象形成树型结构，以表示部分-整体关系的层次结构。
组合模式让客户端可以统一对待单个对象和组合对象。

## 装饰模式
动态地给对象增加一些额外的职责；
拓展功能，比子类更加灵活；

## 外观模式
为子系统的一组接口增加一个统一的入口。
通过定义高层接口，让子系统更加容易使用；

## 享元模式
运用共享技术有效地支持大量细粒度对象的复用
